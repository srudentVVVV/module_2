# Класифікація об'єктів за допомогою мережі Кохонена (SOM)

**Версія:** 1.0.2

## Опис

Десктопний додаток для класифікації об'єктів на основі самоорганізаційної карти Кохонена (Self-Organizing Map). Використовує датасет Iris для демонстрації роботи нейронної мережі.

## Реалізовані функції

- **Мережа Кохонена** — повна реалізація алгоритму SOM: ініціалізація ваг, пошук BMU, оновлення ваг із затуханням learning rate та радіусу сусідства
- **Графічний інтерфейс** — Tkinter GUI з панеллю параметрів, вкладками візуалізації, журналом подій та статус-баром
- **Візуалізація** — U-matrix, Hit map, карта класів, графік помилки квантизації по епохах (matplotlib)
- **Валідація введення** — перевірка числових полів з повідомленнями про помилки
- **Логування** — запис дій та помилок у файл `output/app.log`
- **Конфігурація** — зовнішній файл `config.json` (версія, кольори, шляхи, параметри за замовчуванням)
- **Звіти** — збереження графіків у PNG та текстового звіту з метриками в каталог `output/`
- **Дослідження параметрів** — зміна параметрів мережі через GUI для порівняння результатів
- **Трекер експериментів** — автоматичний запис кожного навчального запуску (параметри, точність, помилка, час) у `output/experiments.json`; вкладка «Експерименти» з таблицею, підсвіткою найкращого результату та експортом у CSV

## Конфігурація

Файл `config.json` містить:
- `version` — версія програми
- `theme_color`, `accent_color` — кольори інтерфейсу
- `output_dir` — каталог для збереження результатів
- `log_file` — шлях до файлу логу
- `default_params` — параметри мережі за замовчуванням (розмір карти, епохи, learning rate, радіус)

## Діаграми

### Діаграма компонентів

![Діаграма компонентів](docs/Діаграма%20компонентів.png)

Діаграма відображає взаємодію між усіма модулями системи. Точка входу `main.py` ініціалізує `src/gui.py` (графічний інтерфейс), який координує роботу решти компонентів: `src/data_loader.py` забезпечує завантаження датасету Iris, `src/kohonen.py` реалізує алгоритм SOM, `src/report.py` відповідає за генерацію графіків і звітів, `src/config_manager.py` читає і зберігає налаштування з `config.json`. Усі модулі записують події через `src/logger_setup.py` у `output/app.log`. Пакет `tests/*.py` тестує кожен модуль незалежно.

### Блок-схема узагальненого алгоритму роботи програми

![Блок-схема узагальненого алгоритму роботи програми](docs/Блок-схема%20узагальненого%20алгоритму%20роботи%20програми.png)

Після старту програма ініціалізує `ConfigManager` і логер, відкриває головне вікно GUI та завантажує датасет Iris. Якщо дані не вдалося завантажити — виводиться повідомлення про помилку і програма завершується. При успішному завантаженні програма переходить у режим очікування подій користувача і обробляє чотири дії:
- **Навчити** — перевіряє валідність параметрів, за потреби показує помилку, інакше запускає алгоритм навчання SOM;
- **Класифікувати** — перевіряє, чи навчена мережа, і виконує класифікацію;
- **Зберегти звіт** — генерує текстовий звіт і PNG-графіки в каталог `output/`;
- **Вихід** — завершує роботу програми.

### Блок-схема алгоритму навчання мережі Кохонена

![Блок-схема алгоритму навчання мережі Кохонена](docs/Блок-схема%20алгоритму%20навчання%20мережі%20Кохонена.png)

Алгоритм навчання SOM отримує параметри `X`, `epochs`, `lr`, `radius` та ініціалізує матрицю ваг `W` випадковими значеннями. На кожній епосі обчислюється поточний коефіцієнт затухання `lr` та `radius`, індекси зразків перемішуються, а потім для кожного зразка `x_i`: знаходиться найближчий нейрон (BMU), розраховується функція сусідства, ваги оновлюються за формулою `W = W + lr × influence × diff`. Після всіх епох встановлюється прапор `is_trained = True`.

### Блок-схема алгоритму класифікації та генерації звіту

![Блок-схема алгоритму класифікації та генерації звіту](docs/Блок-схема%20алгоритму%20класифікації%20та%20генерації%20звіту.png)

При запуску класифікації спочатку перевіряється прапор `is_trained`. Якщо мережа не навчена — виводиться повідомлення про помилку. Інакше обчислюються U-matrix і Hit map, визначаються домінуючі класи для кожного нейрона, розраховується точність моделі та метрики відображаються у GUI. Якщо користувач натискає «Зберегти звіт», програма перевіряє або створює каталог `output/`, генерує PNG-графіки через Matplotlib, формує файл `report.txt` і повідомляє про успішне збереження.
