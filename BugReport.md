## 2. Bug Reports

### Bug Report #1: Shallow Copy мутує DEFAULT_CONFIG

| Поле | Значення |
|------|----------|
| **ID** | BUG-001 |
| **Заголовок** | ConfigManager: shallow copy DEFAULT_CONFIG дозволяє мутацію глобальної константи |
| **Серйозність** | Середня (Medium) |
| **Пріоритет** | Високий (High) |
| **Модуль** | `src/config_manager.py` |
| **Версія** | 1.0.0 |
| **Автор** | Тестувальник |
| **Дата** | 2026-03-27 |
| **Середовище** | Python 3.12, Linux |
| **Передумови** | Програма запущена, файл config.json відсутній |
| **Кроки відтворення** | 1. Створити `ConfigManager` без файлу (генерує конфіг за замовчуванням). 2. Змінити `cm.config["default_params"]["map_rows"] = 999`. 3. Перевірити `DEFAULT_CONFIG["default_params"]["map_rows"]`. |
| **Очікуваний результат** | `DEFAULT_CONFIG["default_params"]["map_rows"]` = 10 (не змінюється) |
| **Фактичний результат** | `DEFAULT_CONFIG["default_params"]["map_rows"]` = 999 (мутовано!) |
| **Причина** | `dict.copy()` створює поверхневу копію — вкладений словник `default_params` спільний для обох об'єктів |
| **Виправлення** | Замінити `DEFAULT_CONFIG.copy()` на `copy.deepcopy(DEFAULT_CONFIG)` у рядках 37 та 40 файлу `config_manager.py` |
| **Тест** | TC-C10 |

### Bug Report #2: Неправильна кількість кольорів у plot_label_map

| Поле | Значення |
|------|----------|
| **ID** | BUG-002 |
| **Заголовок** | ReportGenerator: plot_label_map створює colormap з зайвим кольором |
| **Серйозність** | Низька (Low) |
| **Пріоритет** | Середній (Medium) |
| **Модуль** | `src/report.py` |
| **Версія** | 1.0.0 |
| **Автор** | Тестувальник |
| **Дата** | 2026-03-27 |
| **Середовище** | Python 3.12, matplotlib 3.10, Linux |
| **Передумови** | Мережа навчена, дані Iris (3 класи) |
| **Кроки відтворення** | 1. Навчити SOM. 2. Побудувати label_map. 3. Викликати `plot_label_map(label_map, ["setosa","versicolor","virginica"])`. 4. Перевірити кількість кольорів у colormap. |
| **Очікуваний результат** | Colormap має 4 кольори: 1 (порожній) + 3 (класи) |
| **Фактичний результат** | Colormap має 5 кольорів: 1 (порожній) + 4 (зайвий колір) |
| **Причина** | Рядок 64: `colors = [...][:n_classes + 1]` бере n_classes+1 кольорів замість n_classes. Потім рядок 65 додає ще "#ecf0f1", що дає n_classes+2 замість n_classes+1 |
| **Виправлення** | Змінити `[:n_classes + 1]` на `[:n_classes]` у рядку 64 файлу `report.py` |
| **Тест** | TC-R10 |
